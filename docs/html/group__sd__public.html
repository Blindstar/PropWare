const sd_file_mode  mode

Open a file with a given name and load its information into the file pointer.

Load the first sector of a file into the file buffer; Initialize global character pointers; NOTE: currently, only one file mode is supported and is best described as "r+"; NOTE: two position pointers are used, one for writing and one for reading, this may be changed later to comply with POSIX standards but is useful for my own purposes at the moment NOTE: This driver does not include any provision for timestamps; Niether file modification or creation will changed file's timestamp data (creation times are random and uninitialized)

Precondition:
Files cannot be created in the root directory of a FAT16 filesystem TODO: Fix this
Parameters:
*nameC-string containing the filename to open
*fAddress where file information (such as the first allocation unit) can be stored. Multiple files opened simultaneously is allowed.
Returns:
Returns 0 upon success, error code otherwise
